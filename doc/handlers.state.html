Namespace: state

handlers. state

Handles WAI-ARIA states. This differs from handlers.property in that values are coerced into booleans before being set and a boolean (or the string "mixed") will be returned. handlers.state.read converts the value into a boolean. handlers.state.set sets the state. handlers.state.get gets the state.

(private, static) get(element, name) → {ARIA_state}

Reads the WAI-ARIA state on element. This differs from handlers.property.get in that the result is converted into a boolean or the strign `"mixed"` before being returned. The name is still normalised (see jQuery.normaliseAria).

(private, static) read(raw) → {Boolean|String}

Examples

Converting values

handlers.state.read(true);    // -> true
handlers.state.read("false"); // -> false
handlers.state.read("1");     // -> true
handlers.state.read(0);       // -> false
handlers.state.read("mixed"); // -> "mixed"

Unrecognised values default to true

handlers.state.read("2");      // -> true
handlers.state.read(-1);       // -> true
handlers.state.read([]);       // -> true
handlers.state.read("mixed."); // -> true

(private, static) set(element, name, state, index)

Sets the WAI-ARIA state defined in name on the given element. This differs from handlers.property.set in that state is converted into a boolean or "mixed" before being assigned (see handlers.state.read) and there is no convert paramter. The name is still normalised (see jQuery.normaliseAria).
Parameters:
Name Type Description
element Element Element to modify.
name String WAI-ARIA attribute to set.
state ? State to set.

Example
// Markup is:
// <div id="one"></div>

var one = document.getElementById("one");
var two = document.getElementById("two");

handlers.state.set(two, "checked", "mixed");
